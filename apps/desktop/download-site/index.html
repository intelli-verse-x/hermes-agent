Eligibility and guided first run

  1. Confirm access first. Sign-in requires an admin-authorized work email with an active IVX Agency portal entitlement. The email code expires after 5 minutes; the signed-in session lasts up to 12 hours.
  2. Install the VPN backend before first run: macOS/Linux require wireguard-tools; Windows requires WireGuard for Windows. VPN connection may show an OS-admin password or UAC prompt.
  3. Provisioning is guided after sign-in. The app checks portal entitlement, retrieves only credentials assigned to that user, imports the VPN profile, tests VPN and LiteLLM, and initializes the local agent runtime. Each step can fail independently; open Settings → Connect to inspect status, retry, or enter an administrator-supplied override.
  4. Credential boundary: gateway, model-service, VPN, and confidential service-client credentials are tenant-scoped, stored through operating-system protected storage where supported, and remain revocable by the issuing administrator. Public OAuth clients do not have a client secret.
  5. Support boundary: no response-time SLA is included with early access. Redact tokens, personal data, and local paths before sending logs to support@intelli-verse-x.ai.

Install QuizVerse Desktop

macOS

  1. Download the QuizVerse macOS .dmg from this page.
  2. Open it and drag QuizVerse to Applications, then launch it from Applications.
  3. If macOS cannot verify the developer or notarization ticket, do not bypass the warning. Stop and request a signed, notarized build from your administrator.

Windows

  1. Download the .exe installer; use the MSI only when your organization requires it.
  2. Run the installer, choose the install location if prompted, and start QuizVerse from the Start menu.
  3. If SmartScreen cannot verify the expected Intelliverse X publisher, do not run the installer; request a verified build from your administrator.

Ubuntu / Linux

  1. For the simplest portable install, download the AppImage and run chmod +x QuizVerse-*.AppImage.
  2. Launch the AppImage, or install the .deb on Ubuntu/Debian. An RPM is also provided for compatible distributions.
  3. Desktop integration and microphone prompts vary by distribution and desktop environment.

Updates

QuizVerse checks its own release feed at launch and every four hours. Windows EXE and Linux AppImage installs can download and restart in place. macOS in-place updates require a signed build; package installs such as MSI, deb, and RPM may open this page so you can install the new package manually.

First launch and setup

  1. Let the local agent finish starting. The status strip shows the agent, TutorX, player tools, and available updates. A first launch can take longer while the local runtime is prepared.
  2. Open Setup. Choose hosted TutorX for the managed service, or Local to run TutorX on your computer.
  3. For local TutorX, Python 3.11 or newer and a LiteLLM-compatible API key are required. Use Install TutorX, save the LiteLLM URL/key, run Test LiteLLM, then start TutorX. You can instead select an existing TutorX workspace.
  4. For hosted TutorX, use the service URL supplied by your organization. Some deployments also require a TutorX API key.
  5. Connect your QuizVerse account when your administrator has supplied the Cognito domain, public app client ID, and issuer. The app opens secure browser sign-in and stores resulting tokens in the operating-system keychain. Guest play remains available on surfaces marked for guests; account-only features stay locked until sign-in.
Service prerequisites: internet access is required for sign-in, hosted TutorX, account data, competitions, generated content, realtime matches, and AI requests. Organization-provided Cognito and model-service credentials are not bundled into the installer. Do not paste credentials into chat.

What you can do

TutorX

Use Tutor, Knowledge, Memory, and Learning Space for guided study, knowledge work, review, and progress. Local mode keeps the TutorX workspace on this computer; hosted mode uses your configured service.

Native Play

Open Play for native player experiences. Words includes daily word, groups, spell, crossword, imposter, and vocabulary duel modes. Voyage is a six-planet daily workout with progress, hints, cooldowns, and entitlement-aware premium activities.

Tournaments

Browse eligible competitions, inspect rules, enter through supported paths, play server-issued packs, follow brackets and leaderboards, and claim available certificates. Account, age, eligibility, KYC, balance, and regional rules can limit actions.

Link & Play

Create learning notes from supported URLs, text, documents, images, or audio; then use generated quizzes, flashcards, grounded note chat, spaced repetition, debates, mind maps, speed reading, audio, and other enabled generation tools.

Chat and player tools

Chat can read player-scoped QuizVerse and TutorX information through the bundled player-tool connection. It cannot use agency or administrator access. The Setup lamp reports whether these tools are ready and whether they are operating as a guest or connected account.

Permissions and confirmations

The app asks for microphone access only for voice features. Product writes, tournament changes, game actions, and value-bearing actions show a confirmation dialog. Review the action and payload; cancel anything you did not request. Read-only activity does not require the same confirmation.

Offline use and content cache

  • Some native Words and Voyage routes have deterministic local fallback content and can continue without a connection.
  • Published Words datasets are integrity-checked and kept in an encrypted, expiring cache. A fresh cached copy may be used when the service is temporarily unreachable; the UI identifies offline or fallback provenance.
  • Account synchronization, new content generation, TutorX remote mode, realtime play, entitlements, purchases, leaderboards, and competitions require their services. Work that has not synchronized can remain local until connectivity returns.
  • Local TutorX can keep its workspace on-device, but model calls still require the configured model endpoint unless your organization provides a fully local compatible service.

Troubleshooting and support

Status lights

  • Agent not connected: restart the app and allow first-run setup to finish.
  • TutorX off/error: open Setup, verify Local/Remote mode, test the model endpoint, inspect the visible TutorX log tail, and use Restart.
  • Player tools offline: click Check in Setup, then reconnect the QuizVerse account if account-only tools fail.

Common fixes

  • Confirm the computer clock is correct before troubleshooting sign-in or expired sessions.
  • Allow QuizVerse through firewalls/proxies for the configured TutorX, Cognito, model, QuizVerse API, and realtime service endpoints.
  • For microphone failures, grant QuizVerse access in the operating system's privacy settings, then restart the app.
  • If an update cannot install in place, download the current installer from this tab and install over the existing copy.

Logs and support

Use the app's Open logs action when available. The main diagnostic file is desktop.log in the isolated local profile's logs directory under the QuizVerse app-data folder; player-tool audit events are in quizverse-mcp-audit.jsonl there. Remove access tokens, personal data, local paths, and uploaded content before sharing logs. Contact support@intelli-verse-x.ai with the OS, app version, failing tab, time of failure, and a redacted excerpt.

Privacy and security

  • QuizVerse uses an app-specific data directory and isolated local agent profile; it does not import IVX Admin Desktop credentials or settings.
  • Account and model-service secrets are encrypted with the operating system's secure storage and are not returned to the settings screen after saving.
  • Player tools are restricted to the current player's allowed QuizVerse operations. Writes use explicit confirmation, short-lived approval challenges, and idempotency protection; audit records stay local.
  • Files submitted to Link & Play or other online generation features are sent to the configured QuizVerse service. Review your organization's privacy and retention policy before uploading sensitive material.
  • Install only artifacts exposed by this page and verify checksums shown beneath a download when required.

Uninstall and local data

Typical app-data locations are:

  • macOS: ~/Library/Application Support/QuizVerse
  • Windows: %APPDATA%\QuizVerse
  • Linux: ~/.config/QuizVerse

The folder contains settings, the isolated local agent profile, logs, cached Words content, local TutorX workspace and managed environment (if installed), and browser/session storage. Uninstall QuizVerse through Applications, Windows Settings, or your package manager. Standard OS uninstall may leave app data so a reinstall can recover it. To remove everything, sign out, quit QuizVerse, back up any local TutorX work you need, uninstall the app, and then delete only the QuizVerse app-data folder listed above. Never delete the IVX Agency data folder.
